MolProbity often relies on symbolic links (symlinks), so it is critical that your webserver be configured to follow symlinks fot the MolProbity directories. For Apache, you should have a Options FollowSymLinks directive in the proper part of your httpd.conf file (usually in /etc/httpd/httpd.conf).

(Is this bit about symlinks still true?)

In order for the web server to be able to access MolProbity data correctly, it must have read, write, and execute permissions on the public_html/data/ and tmp/ directories. If someone other than the web server pseudo-user owns the files, this probably implies a permission mask of 777. The same thing goes for public_html/data/molprobity.log.

It's also important to be sure the programs in the bin/ directory have their execute bit set for the web server pseudo-user.

It's critical that the absolute path to the directory where MolProbity is installed not have any spaces or other funky characters in it. Alphanumerics, underscores, dashes, and dots are OK. Everything else is off limits. Those weird characters interfere both with passing paths in URLs (they have to be encoded and decoded) and with UNIX command line programs (all paths have to be enclosed in extra quote marks).
